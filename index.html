First: What is an API

There are several ways one could describe what an API is.

Simply put, an API is like a third party buffer that allows a user to access a data collection, service, software, etc, that is available publicly or privately. It sets up a set of rules and documentation for a user to work with so they in turn could build applications based on said service or data set. In my example, we will be using the BrewersDB site to demonstrate how to write GET requests to retrieve a variety of information from the BrewersDB collection. Many might write a website or mobile APP that would be richer with features. My aim is that this how-to-guide would help a user do so.

Developers typically include documentation with their API for users to write their applications around. Below is a link (new window) to the breweryDB's API.

BreweryDB API

Furthermore, to use an API, you will need to get a key from the developer. Below is a link (new window), to get a key. You will first need to register with BreweryDB and will very shortly be issued a key to work with the API.
