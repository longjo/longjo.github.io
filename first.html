Constructing a GET Request

Your First GET Request...

Now that we have explained a bit what an API is and specifically what you can do with the BrewersDB API, we will write our first GET request.

Here is what a typical GET request looks like for brewersDB:

https://api.brewerydb.com/v2/beers?key=MYKEY&styleId=12


Lets Break this down a bit...

There are three parts to this GET request...

The First Part:

https://api.brewerydb.com/v2/beers?

This is saying that we want to access the Beer(s) endpoint. A premium member would be able to append their key to this and get a list of every beer listed with BrewersDB. Without any doubt, the list would be exhaustive.

The Second Part:

key=MYKEY

This is where you append your key to the GET request. Ideally you want to keep your key private, so no one can misuse your key, so you alone have access to the it.

The Third Part:

&styleId=12

This is a request parameter that we used in our query to further specify a search result.

What Our Get Request Is Returning:

When this request is executed, we will get as a response a JSON listing all beers that have styleID as 12.

styleID 12 corresponds to: "English Style Brown Ale".
